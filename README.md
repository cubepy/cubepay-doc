## این پروژه چیه؟

CubePay یه **سیستم چند-فروشنده‌ایه** برای پرداخت کارت‌به‌کارت که به‌جای تکیه به یه درگاه بانکی رسمی، از **تشخیص خودکار پیامک بانکی** استفاده می‌کنه. هر فروشنده کارت خودش رو ثبت می‌کنه، مشتری مستقیم بهش واریز می‌کنه، و یه گوشی فیزیکی همون پیامک بانک رو به سرور فوروارد می‌کنه تا تراکنش خودکار تایید بشه.

### 🧩 چیزی که CubePay رو متفاوت می‌کنه

| |  |
|---|---|
| 🔄 **چرخش چندکارتی** | چند کارت ثبت کن، بذار بین‌شون دستی/رندوم/ساعتی بچرخه — دیگه یه کارت تنها بار همه‌ی تراکنش‌ها رو نمی‌کشه |
| 🏦 | نمایش شبا در کنار کارت، برای انتقال آنیِ اختیاری |
| 🎁 **بونوس پلکانی کیف‌پول** | هرچی بیشتر شارژ کنی، درصد بیشتری بونوس می‌گیری (تا ۲۰٪) |
| 🤖 **مدیریت کامل از تو تلگرام** | ثبت‌نام، تایید، کارت، کارمزد، کیف‌پول، خروجی اکسل — همه از تو چت، بدون پنل وب جدا |
| 🧾 **فاکتور دستی** | بدون نوشتن حتی یک خط کد، از تو ربات یه لینک پرداخت بساز |
| 📲 **بدون درگاه بانکی، بدون نماد** | فقط یه کارت به نام خودت کافیه |

> ## 🚨🚨🚨 هشدار حیاتی — قبل از هر چیز بخونید 🚨🚨🚨
>
> ### 📵 گوشیِ فورواردر پیامک باید **همیشه، ۲۴ ساعته، بدون وقفه** به اینترنت وصل باشه.
>
> این سیستم کاملاً به **وب‌هوک** متکیه: پیامک بانکی روی یه گوشی فیزیکی دریافت می‌شه و از طریق اینترنت همون گوشی به سرور فرستاده می‌شه.
>
> **اگه گوشی خاموش باشه، اینترنتش قطع باشه، یا اپ فورواردر بسته/کشته شده باشه:**
> - ❌ پیامک بانک اصلاً به سرور نمی‌رسه
> - ❌ تراکنش تشخیص داده نمی‌شه
> - ❌ مشتری علی‌رغم واریز واقعی، تاییدیه نمی‌گیره
>
> **این مسئولیت مستقیماً با فروشنده‌ست** — نه باگ سیستم، نه مشکل CubePay.
>
> ✅ توصیه‌های ضروری:
> - همیشه شارژر به گوشی وصل باشه (یا به پریز برق دائم وصل باشه)
> - بهینه‌سازی باتری (Battery Optimization) برای اپ فورواردر **غیرفعال** باشه
> - از یه سیم‌کارت با اینترنت پایدار و بدون قطعی استفاده کنید
> - هر از گاهی خودتون تست کنید که سیستم هنوز جواب می‌ده

## ⚡️ سریع‌ترین راه شروع — بدون نیاز به فهمیدن API

اگه نمی‌خواید API رو دستی صدا بزنید، فایل آماده‌ی `CubePayClient.php` رو بردارید — فقط توکن و دامنه رو پر کنید:

```php
require 'CubePayClient.php';
$cubepay = new CubePayClient('YOUR_API_TOKEN', 'https://YOUR-CUBEPAY-DOMAIN/smspay');

$result = $cubepay->createPayment(200000, 'ORDER123', 'https://yourbot.example.com/callback.php');
if ($result['success']) {
    echo $result['payment_link'];
}
```

نمونه‌ی کامل ساخت فاکتور: [`examples/create-payment-example.php`](examples/create-payment-example.php)
نمونه‌ی کامل دریافت کال‌بک: [`examples/callback.php`](examples/callback.php)

همین کافیه — نیازی به Composer، کتابخونه‌ی اضافه، یا خوندن بقیه‌ی این صفحه نیست. بقیه‌ی داکیومنت برای کسانیه که می‌خوان مستقیم با API کار کنن (مثلاً با Python/Node) یا جزئیات بیشتری بخوان.

## 📌 نکات مهم

✔ مبلغ‌ها همه‌جا به **ریال** ارسال/دریافت می‌شن (تومان × ۱۰)
✔ فعلاً فقط `type: "card"` پشتیبانی می‌شه
✔ همیشه از `pay_amount_toman` برای نمایش مبلغ نهایی به مشتری استفاده کنید، نه مبلغ خام درخواستی
