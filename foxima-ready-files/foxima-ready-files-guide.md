# 📦 اتصال آماده‌ی CubePay به ربات Foxima (نصب با فایل، نه ویرایش دستی)

</div>

این راهنما برای فروشنده‌هایی است که ربات فروش‌شان را با **Foxima** (یا فورک‌های آن) ساخته‌اند. به‌جای این‌که خودتان فایل‌ها را به‌صورت دستی ویرایش کنید، فقط فایل‌های آماده را جایگزین کنید؛ سریع‌تر، ساده‌تر و با احتمال خطای کمتر.

> **🔗 پروژه اصلی Foxima**
>
> https://github.com/Mmd-Amir/Faoxima
>
> این فایل‌های آماده بر اساس ساختار همین پروژه تهیه شده‌اند و با اکثر فورک‌های آن نیز سازگار هستند.

> ⚠️ **قبل از شروع:** حتماً از پوشه‌ی ربات خود یک نسخه‌ی پشتیبان (Backup) تهیه کنید. فایل‌های این بسته به‌صورت کامل جایگزین فایل‌های فعلی می‌شوند؛ بنابراین اگر قبلاً تغییراتی روی همین فایل‌ها داده باشید، آن تغییرات از بین خواهند رفت.

---

# 🔴 بخش ۱ — فایل‌های اجرایی (اجباری)

بدون این دو فایل، CubePay اصلاً کار نخواهد کرد.

| فایل                   | مسیر جایگزینی                         |
| ---------------------- | ------------------------------------- |
| `business_logic_1.php` | `re/rx/function/business_logic_1.php` |
| `successful.php`       | `payment/ZarinPay/successful.php`     |

این دو فایل مسئول ایجاد فاکتور پرداخت و تأیید خودکار تراکنش هستند.

**هیچ اطلاعات شخصی (توکن یا رمز) داخل این فایل‌ها وجود ندارد.** توکن API را بعداً از داخل پنل ربات خودتان وارد خواهید کرد.

---

# 🔵 بخش ۲ — فایل‌های ظاهری و پنل (اختیاری)

این فایل‌ها فقط ظاهر پنل را تغییر می‌دهند و برای عملکرد CubePay الزامی نیستند.

| فایل                   | مسیر جایگزینی                  |
| ---------------------- | ------------------------------ |
| `bootstrap_1.php`      | `re/rx/admin/bootstrap_1.php`  |
| `bootstrap_2.php`      | `re/rx/admin/bootstrap_2.php`  |
| `layouts_1.php`        | `re/rx/keyboard/layouts_1.php` |
| `settings.php`         | `re/rx/admin/settings.php`     |
| `service_keyboard.php` | `panel/service_keyboard.php`   |

این فایل‌ها فقط نام «زرین‌پی» را در پنل مدیریت و رابط کاربری به «کیوب‌پی» تغییر می‌دهند و هیچ تغییری در منطق پرداخت ایجاد نمی‌کنند.

اگر برایتان مهم نیست که هنوز نام «زرین‌پی» در پنل نمایش داده شود، می‌توانید این بخش را نصب نکنید و CubePay همچنان بدون مشکل کار خواهد کرد.

### نکته درباره `bootstrap_2.php`

این فایل علاوه بر تغییر متن‌ها، دکمه‌ی **«آموزش اتصال»** را نیز تغییر می‌دهد.

در نسخه‌ی اصلی Foxima این دکمه لینک تماس سازنده‌ی اولیه‌ی زرین‌پی را نمایش می‌دهد. اگر نمی‌خواهید کاربران شما آن لینک را ببینند، پیشنهاد می‌شود حداقل همین فایل را جایگزین کنید.

### یک مرحله SQL (در صورت نصب بخش ۲)

اگر فایل‌های بخش ۲ را نصب می‌کنید، این دستور را نیز یک‌بار در phpMyAdmin (تب SQL) اجرا کنید:

```sql
UPDATE textbot
SET text = '🟠 کیوب پی'
WHERE id_text = 'zarinpey';
```

# ✅ مراحل نصب

## قدم ۱ — تهیه نسخه پشتیبان

از کل پوشه‌ی ربات خود یک Backup تهیه کنید.

---

## قدم ۲ — جایگزینی فایل‌ها

فایل‌های موردنیاز را دقیقاً در مسیرهای گفته‌شده جایگزین کنید.

---

## قدم ۳ — ثبت‌نام در CubePay

اگر هنوز فروشنده نشده‌اید، وارد ربات زیر شوید:

**@cubepy_bot**

بعد از ثبت‌نام و تأیید حساب، از بخش **«🔗 پنل من»** توکن API خود را دریافت کنید.

---

## قدم ۴ — وارد کردن توکن در Foxima

در پنل مدیریت ربات بروید به:

```text
پنل مدیریت
→ درگاه‌های ریالی
→ زرین‌پی
```

(اگر بخش ۲ را نصب کرده باشید، نام آن «کیوب‌پی» خواهد بود.)

تنظیمات:

* ✅ فعال‌سازی: روشن
* 🔑 توکن API: توکن دریافت‌شده از CubePay
* (اختیاری) حداقل مبلغ
* (اختیاری) حداکثر مبلغ
* (اختیاری) کش‌بک

---

## قدم ۵ — افزودن کارت بانکی و اتصال پیامک

در **@cubepy_bot**:

* **💳 مدیریت کارت‌ها** → کارت بانکی خود را اضافه کنید.
* **📲 آموزش اتصال پیامک** → اپلیکیشن Forwarder پیامک را طبق آموزش روی گوشی نصب و تنظیم کنید.

> ⚠️ گوشی‌ای که اپلیکیشن روی آن نصب شده است باید همیشه به اینترنت متصل باشد؛ در غیر این صورت پرداخت‌ها به‌صورت خودکار تأیید نخواهند شد.

---

## قدم ۶ — تست نهایی

یک پرداخت آزمایشی (مثلاً ۱۰٬۰۰۰ تومان) انجام دهید.

اگر همه‌چیز درست باشد:

1. لینک پرداخت باز می‌شود.
2. مبلغ صحیح نمایش داده می‌شود.
3. مشتری کارت‌به‌کارت انجام می‌دهد.
4. چند ثانیه بعد پرداخت به‌صورت خودکار تأیید می‌شود.
5. سفارش یا شارژ کیف پول به‌صورت خودکار تحویل داده می‌شود.

---

# ❓ اگر فایل‌ها کار نکردند

اگر بعد از جایگزینی فایل‌ها با خطا مواجه شدید، احتمالاً نسخه‌ی Foxima شما با نسخه‌ای که این فایل‌ها بر اساس آن تهیه شده‌اند متفاوت است یا قبلاً تغییراتی روی همان فایل‌ها انجام داده‌اید.

در این حالت:

* نسخه‌ی پشتیبان را برگردانید.
* به‌جای استفاده از فایل‌های آماده، از **راهنمای ویرایش دستی** (`foxima-integration-guide.md`) استفاده کنید. در آن راهنما فقط چند خط مشخص از فایل‌های خودتان تغییر می‌کند و نیازی به جایگزینی کامل فایل‌ها نیست.

اگر همچنان مشکل داشتید، فایل `error_log` کنار فایل‌های PHP را برای بررسی ارسال کنید.
