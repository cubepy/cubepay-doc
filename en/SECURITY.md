# Security Policy

## Reporting a Vulnerability

If you find a security issue (not a regular bug) in the CubePay API or
infrastructure, **please do not open a public issue.**

Instead, report it through:

- A direct, private message to [@cubepy_bot](https://t.me/cubepy_bot)

## What should you report?

- Unauthorized access to other users' transaction data
- Bypassing the authorization mechanism
- Ability to forge or replay a callback without authentication
- Sensitive information disclosed in API responses

## What you don't need to report

- Ordinary validation errors (e.g. 422 error messages)
- Feature requests → use [CONTRIBUTING.md](./CONTRIBUTING.md) instead

Thank you for responsibly helping keep CubePay users safe.
