# 📦 اتصال آماده‌ی CubePay به ربات Foxima (نصب با فایل، نه ویرایش دستی)

این راهنما برای فروشنده‌هاییه که ربات فروش‌شون رو با کد **Foxima** (یا فورک‌های مشابه) ساختن. به‌جای این‌که خودتون خط‌به‌خط کد رو ویرایش کنید، فقط این فایل‌های آماده رو جایگزین کنید — سریع‌تر و بی‌خطاتر.

⚠️ **قبل از شروع:** یه نسخه پشتیبان (بک‌آپ) از پوشه‌ی ربات‌تون بگیرید. این فایل‌ها **کامل جایگزین** فایل‌های فعلی می‌شن؛ اگه قبلاً خودتون تغییری تو همین ۵ فایل داده باشید، از دست می‌ره.

---

## 🔴 بخش ۱ — فایل‌های اجرایی (اجباری، بدون این‌ها CubePay اصلاً کار نمی‌کنه)

| فایل | مسیر جایگزینی |
|---|---|
| `business_logic_1.php` | `re/rx/function/business_logic_1.php` |
| `successful.php` | `payment/ZarinPay/successful.php` |

این دو فایل، **قلب اتصال** به CubePay‌ان — ساخت فاکتور و تایید پرداخت رو انجام می‌دن. هیچ اطلاعات شخصی‌ای (توکن، رمز) توشون نیست؛ توکن رو جدا از تو ربات (توضیح در ادامه) وارد می‌کنید.

---

## 🔵 بخش ۲ — فایل‌های ظاهری/کیبورد (اختیاری، فقط برای حذف اسم «زرین‌پی»)

| فایل | مسیر جایگزینی |
|---|---|
| `bootstrap_1.php` | `re/rx/admin/bootstrap_1.php` |
| `bootstrap_2.php` | `re/rx/admin/bootstrap_2.php` |
| `layouts_1.php` | `re/rx/keyboard/layouts_1.php` |
| `settings.php` | `re/rx/admin/settings.php` |
| `service_keyboard.php` | `panel/service_keyboard.php` |

این پنج‌تا فقط **متن دکمه‌ها** (تو چت ادمین ربات و تو پنل وب) رو از «زرین پی» به «کیوب پی» تغییر می‌دن — هیچ منطقی رو عوض نمی‌کنن. اگه اسم «زرین پی» تو پنل ادمین خودتون براتون مهم نیست، می‌تونید این ۵ فایل رو اصلاً جایگزین نکنید — CubePay بدون‌شون هم درست کار می‌کنه.

📌 **یه نکته‌ی جدا از اسم:** فایل `bootstrap_2.php` یه چیز دیگه هم داره — دکمه‌ی «آموزش اتصال» تو نسخه‌ی اصلی Foxima، پیام و **لینک تماس شخصی سازنده‌ی اولیه‌ی زرین‌پی** رو نشون می‌ده (نه شما، نه CubePay). اگه دوست ندارید مشتری‌هاتون یه لینک تماس ناشناس ببینن، پیشنهاد می‌کنم این یکی رو حتی اگه به اسم «زرین‌پی» حساس نیستید، جایگزین کنید.

📌 **یه قدم SQL هم لازمه** (اگه بخش ۲ رو جایگزین می‌کنید): یه اسم دیگه‌ی «زرین پی» تو خود دیتابیس ذخیره شده (نه تو فایل)، که هم رو دکمه‌ی خرید مشتری هم رو یه لیست تو پنل وب اثر می‌ذاره. این یه خط رو تو phpMyAdmin (تب SQL) بزنید:
```sql
UPDATE textbot SET text = '🟠 کیوب پی' WHERE id_text = 'zarinpey';
```

## ✅ قدم‌های نصب

### قدم ۱ — بک‌آپ
از پوشه‌ی ربات‌تون یه کپی بگیرید.

### قدم ۲ — جایگزینی فایل‌ها
بسته به این‌که فقط بخش ۱ رو می‌خواید یا هر ۵ تا رو، فایل‌ها رو دقیقاً تو همون مسیرهای بالا جایگزین کنید.

### قدم ۳ — ثبت‌نام تو CubePay
اگه هنوز فروشنده نشدید، به [@cubepy_bot](https://t.me/cubepy_bot) پیام بدید و ثبت‌نام کنید (نام کسب‌وکار، شماره تلفن، تایید قوانین). بعد از تایید ادمین، از «🔗 پنل من» توکن API‌تون رو بردارید.

### قدم ۴ — وارد کردن توکن تو ربات خودتون
تو ربات خودتون (Foxima)، برید:
```
پنل مدیریت → درگاه‌های ریالی → زرین‌پی (یا کیوب‌پی، اگه بخش ۲ رو هم جایگزین کردید)
```
و این‌ها رو تنظیم کنید:
- ✅ فعال‌سازی: روشن
- 🔑 توکن: همونی که از CubePay گرفتید
- (اختیاری) حداقل/حداکثر مبلغ، کش‌بک

### قدم ۵ — تنظیم کارت و وب‌هوک پیامک تو CubePay
تو `@cubepy_bot`:
- «💳 مدیریت کارت‌ها» → کارت خودتون رو اضافه کنید
- «📲 آموزش اتصال پیامک» → اپ فورواردر رو رو گوشی خودتون طبق راهنما تنظیم کنید

⚠️ **یادتون نره:** گوشی‌ای که این اپ روشه باید همیشه به اینترنت وصل باشه، وگرنه هیچ تراکنشی تایید نمی‌شه.

### قدم ۶ — تست
یه شارژ کیف‌پول کوچیک (مثلاً ۱۰,۰۰۰ تومان) تو ربات خودتون امتحان کنید. باید:
1. لینک پرداخت باز بشه، مبلغ دقیق (با چند تومان اضافه) نشون بده
2. بعد از کارت‌به‌کارت، چند ثانیه‌ای خودکار تایید بشه
3. ربات‌تون سرویس/شارژ رو خودکار تحویل بده

## ❓ اگه فایل‌ها جواب نداد

اگه بعد از جایگزینی خطا گرفتید، احتمالاً نسخه‌ی Foxima شما با نسخه‌ای که این فایل‌ها ازش گرفته شده فرق داره (مثلاً قبلاً خودتون یه‌جای دیگه‌ی همین فایل‌ها رو دستی تغییر داده بودید). تو این حالت:
- بک‌آپ رو برگردونید
- به‌جای این فایل‌ها، از **راهنمای ویرایش دستی** (`foxima-integration-guide.md`) استفاده کنید — همون ۲ خط مشخص رو تو فایل خودتون تغییر بدید، بدون این‌که کل فایل عوض بشه

اگه بازم گیر کردید، `error_log` کنار فایل‌های PHP رو بفرستید تا دقیق بررسی بشه.
