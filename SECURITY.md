# سیاست امنیتی

## گزارش آسیب‌پذیری

اگه یک مشکل امنیتی (نه یک باگ معمولی) در API یا زیرساخت CubePay پیدا کردید،
**لطفاً از باز کردن Issue عمومی خودداری کنید.**

به‌جاش از این راه گزارش بدید:

- پیام مستقیم و خصوصی به [@cubepy_bot](https://t.me/cubepy_bot)

## چه چیزی رو گزارش کنیم؟

- دسترسی غیرمجاز به داده‌های تراکنش دیگران
- دور زدن مکانیزم Authorization
- امکان جعل یا تکرار Callback بدون تایید هویت
- افشای اطلاعات حساس در پاسخ‌های API

## چه چیزی لازم نیست گزارش بدید؟

- خطاهای معمول اعتبارسنجی (مثل پیام‌های خطای 422)
- درخواست ویژگی جدید → از طریق [CONTRIBUTING.md](./CONTRIBUTING.md)

از همکاری مسئولانه‌ی شما در حفظ امنیت کاربران CubePay سپاسگزاریم.
