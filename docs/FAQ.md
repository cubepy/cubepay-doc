# ❓ سوالات متداول و عیب‌یابی

## سوالات عمومی

**آیا نیاز به درگاه بانکی رسمی دارم؟**
❌ نه، فقط یک کارت بانکی به نام خودتون کافیه.

**فاکتورها چند دقیقه اعتبار دارن؟**
⏳ ۳۰ دقیقه.

**اگه پیامک بانک دیر برسه چی؟**
تا وقتی فاکتور منقضی نشده، تشخیص انجام می‌شه؛ تاخیر پیامک بانک معمولاً چند ثانیه‌ست.

**چطور توکن یا کارتم رو عوض کنم؟**
از منوی ربات مدیریت فروشندگان ([@cubepy_bot](https://t.me/cubepy_bot)).

**چطور Auto Confirmation رو فعال کنم؟**
تایید خودکار به‌صورت پیش‌فرض روی همه‌ی حساب‌های تاییدشده فعاله؛ کافیه کارت بانکی و اپلیکیشن Forwarder پیامک رو درست تنظیم کرده باشید.

**گوشی‌ای که پیامک‌ها رو فوروارد می‌کنه باید همیشه روشن باشه؟**
✅ بله. اگه اون گوشی آفلاین بشه، پیامک‌های بانکی به سیستم نمی‌رسن و تایید خودکار متوقف می‌شه.

## سوالات فنی / زبان‌ها

**آیا از PHP پشتیبانی می‌شه؟**
✅ بله، API عمومی و مبتنی بر HTTP/JSON هست، از هر زبانی قابل استفاده‌ست. نمونه کد در [docs/examples/](./examples/).

**آیا از Node.js پشتیبانی می‌شه؟**
✅ بله، نمونه کد Node.js هم موجوده.

**آیا می‌تونم از SQLite برای ذخیره‌ی تراکنش‌ها استفاده کنم؟**
✅ بله، CubePay هیچ محدودیتی روی دیتابیس سمت شما نمی‌ذاره. شما فقط `order_id` و `authority` رو ذخیره می‌کنید؛ انتخاب دیتابیس (SQLite, MySQL, PostgreSQL, …) کاملاً به خودتون بستگی داره.

**آیا کتابخانه‌ی رسمی (SDK) دارید؟**
فعلاً یک کلاینت نمونه‌ی PHP در [`docs/examples/CubePayClient.php`](./examples/CubePayClient.php) ارائه شده؛ برای بقیه‌ی زبان‌ها از نمونه کدهای HTTP ساده استفاده کنید.

## سوالات مربوط به پلتفرم‌ها

## 🛠 عیب‌یابی (Troubleshooting)

### ❌ خطای 401 Unauthorized
- مطمئن بشید هدر رو دقیقاً به‌شکل `Authorization: Bearer YOUR_API_TOKEN` می‌فرستید (نه فقط توکن خام).
- بررسی کنید فاصله یا کاراکتر اضافه‌ای موقع کپی توکن وارد نشده باشه.
- توکن رو از پنل ربات دوباره چک/رجنریت کنید.

### ❌ Webhook / Callback دریافت نمی‌شود
- آدرس `callback_url` باید **https** معتبر و از بیرون قابل‌دسترس باشه (نه `localhost` یا IP داخلی).
- فایروال یا Cloudflare سمت سرور خودتون رو چک کنید که درخواست POST از سمت CubePay بلاک نشه.
- لاگ سرور خودتون رو برای درخواست‌های دریافتی به مسیر callback بررسی کنید.

### ❌ Connection Timeout
- اتصال شبکه‌ی سرور شما به `cubevps.ir` رو با `curl -I https://cubevps.ir` تست کنید.
- اگه از سرور داخل ایران با تحریم شبکه یا فیلترینگ مواجهید، DNS/مسیر شبکه رو بررسی کنید.

### ❌ SSL Error
- مطمئن بشید نسخه‌ی cURL/OpenSSL سرورتون به‌روز است.
- در محیط توسعه هرگز `CURLOPT_SSL_VERIFYPEER` رو `false` نکنید؛ در عوض CA certificate سیستم رو آپدیت کنید.

### ❌ `verify-payment` همیشه `pending` برمی‌گردونه
- تراکنش هنوز واریز نشده یا پیامک بانکی هنوز پردازش نشده — چند ثانیه صبر کنید و دوباره تلاش کنید (پیش از انقضای ۳۰ دقیقه‌ای فاکتور).
- گوشی متصل به اپلیکیشن Forwarder رو چک کنید که آنلاین و متصل باشه.

### ❌ فایل‌های آماده‌ی Foxima کار نکرد
- نسخه‌ی Foxima شما احتمالاً با نسخه‌ای که فایل‌های آماده بر اساسش تهیه شده فرق داره، یا قبلاً همون فایل‌ها رو شخصی‌سازی کرده بودید.
- از [راهنمای دستی Foxima](../integrations/faoxima-integration-guide.md) استفاده کنید که فقط چند خط مشخص تغییر می‌کنه.

سوالی که جواب پیدا نکردید؟ از طریق [cube_sup](https://t.me/cube_sup) بپرسید.
